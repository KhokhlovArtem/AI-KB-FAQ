## Роли и зоны ответственности

| Роль | Основные задачи |
|------|----------------|
| **Backend-разработчик (Python)** | Orchestrator API, Telegram-бот, интеграция с LLM API, пайплайн индексации, логика ветвления |
| **DevOps / Инженер инфраструктуры** | Развертывание векторной БД, настройка окружения, CI/CD, GitLab интеграция, контейнеризация |
| **ML-инженер / LLM-инженер** | Выбор embedding-модели, настройка параметров поиска, порог релевантности, оптимизация промптов, тестирование качества ответов |

---

## Оценка часов (порядок)

### Backend-разработчик — 80–120 часов

| Задача | Часы |
|--------|------|
| Проектирование архитектуры + FastAPI скелет | 8–12 |
| Telegram-бот (aiogram) + обработка сообщений | 16–24 |
| Интеграция с YandexGPT/GigaChat API (обертка, retry, fallback) | 12–16 |
| Embedding-пайплайн + интеграция с векторной БД | 12–20 |
| Логика поиска + порог релевантности | 8–12 |
| Формирование и отправка email-тикетов | 6–10 |
| Логирование, обработка ошибок, мониторинг | 8–12 |
| Тестирование интеграций, отладка | 10–14 |

### DevOps — 40–60 часов

| Задача | Часы |
|--------|------|
| Развертывание Qdrant / pgvector (k8s / docker-compose) | 8–12 |
| Настройка CI/CD (GitLab CI / GitHub Actions) | 8–12 |
| Контейнеризация приложений (Docker) | 6–8 |
| Интеграция с GitLab API для выгрузки БЗ | 6–10 |
| Настройка переменных окружения, секретов | 4–6 |
| Базовый мониторинг (логи, метрики) | 6–12 |

### ML/LLM-инженер — 40–60 часов

| Задача | Часы |
|--------|------|
| Анализ структуры документов, выбор стратегии чанкинга | 8–12 |
| Подбор embedding-модели + тестирование качества поиска | 10–16 |
| Настройка и эксперименты с порогом релевантности | 8–12 |
| Оптимизация промптов для LLM | 6–10 |
| Оценка качества ответов, итеративные улучшения | 8–12 |

## Итоговая оценка

| Роль | Часы (приблизительно) |
|------|----------------------|
| Backend-разработчик | 80–120 |
| DevOps | 40–60 |
| ML/LLM-инженер | 40–60 |
| **Итого** | **160–240 человеко-часов** |

В переводе на рабочие недели:
- Если над проектом работает **один разработчик** (совмещающий роли) → **4–6 недель** (full-time)
- Если **команда из 3 человек** параллельно → **2–3 недели** календарных

## Дополнительные факторы, которые могут увеличить объем

| Фактор | Влияние |
|--------|--------|
| Объем базы знаний окажется очень большим (10k+ документов) | +20–30% на оптимизацию индексации и пайплайнов |
| Жесткие требования к безопасности (все в закрытом контуре) | +20–30% на DevOps |
| Необходимость поддержки нескольких LLM (и Yandex, и Giga) с переключением | +15–20% на backend |
| Кастомизация email-тикетов (шаблоны, приоритеты, привязка к системе поддержки) | +10–20% |
| Сложная структура документов (не однородная, нужно извлекать метаданные) | +15–25% на ML-часть |
